## Задание

Требования к выполнению работы:

- Тестовое покрытие должно быть сформировано на основании набора прецедентов использования сайта.
- Тестирование должно осуществляться автоматически - с помощью системы автоматизированного тестирования Selenium.
- Шаблоны тестов должны формироваться при помощи Selenium IDE и исполняться при помощи Selenium RC в браузерах Firefox и Chrome.
- Предполагается, что тестируемый сайт использует динамическую генерацию элементов на странице, т.е. выбор элемента в DOM должен осуществляться не на основании его ID, а с помощью XPath.

Сайт: https://sravni.bet/

## Прецеденты 

В этом разделе опишем основные прецеденты использования сайта. 

1. `Открытие главной страницы` - проверка загрузки страницы при заходе
2. `Выбор вида спорта` - пользователь выбирает вид спорта
3. `Просмотр списка матчей` - после выбора спорта пользователь просматривает список матчей
4. `Выбор матча` - пользователь выбирает матч
5. `Проверка отображения коэффиентов` - на странице матча видны коэффициенты от разные брокерских компаний
6. `Переход на сайт букмекера` - пользователь переходит на сайт букмекера
7. `Фильтрация по времени` - переключение по вкладкам "сегодня", "завтра" и т.д.
8. `Поиск` - поиск события/матча

## Selenium

Selenium - автоматизированное тестирование веб-приложений. Для получения шаблона тестов используется Selenium Generate Page Object. Это расширение для браузера, которое генерирует шаблоны тестов на основе URL-адреса.
